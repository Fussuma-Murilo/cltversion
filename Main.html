Gerênciamento de configurações de software

O gerênciamento de configurações, é um processo de engenharia de sistemas essêncial para o desenvolvimento de um software, utilizado para acompanhar alterações realizadas ao decorrer do seu desenvolvimento.

Alguns dos processos utilizados, são -------->

Controle de versões

Esta é uma prática que consiste em armazenar diversas versões do código para que o mesmo possa ser consultado a fim de acompanhamento. Este acompanhamento é realizado pelos próprios desenvolvedores, o que facilita a colaboração.

Controle de mudanças

O controle de mudanças é responsável pelo controle de solicitações de alteração no projeto. Diferentemente do controle de versões, o controle de mudanças é utilizado para acompanhar as solicitações requisitadas durante o desenvolvimento do software, para os mesmos fins.

Integração contínua

A integração contínua é a prática de automatizar a integração do código provido por diversos contribuidores em um único projeto. Permite a agilidade durante o processo de mesclagem do código em um repositório principal. Tendo como um requisito, um bom controle de versões do código fonte.
